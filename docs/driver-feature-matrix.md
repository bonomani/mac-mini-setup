## `pkg` backends in detail

`pkg.sh` is the dominant driver. Its 8 backends share a uniform interface
(`_pkg_<be>_available / _observe / _install / _update / _version / _activate`
and an optional `_outdated`):

| Backend | Implicit dep | Provided-by | Outdated mechanism | Notes |
|---|---|---|---|---|
| `brew` | `homebrew` | `brew` | `brew outdated` always; `brew livecheck` when opt-in flag set | Tap-aware via `_brew_cached_version` strip-prefix |
| `brew-cask` | `homebrew` | `brew-cask` | `brew outdated --cask` (+ `--greedy` if YAML asks) | Reads `driver.greedy_auto_updates` |
| `native-pm` | `build-deps` | `native-package-manager` | per-PM cache: `apt list --upgradable` / `dnf check-update` / `pacman -Qu` / `zypper -n list-updates` | Linux/WSL2 only |
| `npm` | `node-lts` | `npm` | `npm outdated -g --json`, cached | Activates nvm via `_npm_ensure_path` |
| `pyenv` | `pyenv` | `pyenv` | — | Activates shims via `_pyenv_ensure_path` |
| `ollama` | `ollama` | `ollama` | — | Wraps `ollama_model_present` / `ollama_model_pull` |
| `vscode` | `vscode-code-cmd` | `vscode-marketplace` | marketplace `extensionquery` POST API, bulk + cached | Wraps `vscode_extension_install/update` |
| `curl` | (none) | `curl` | `_pkg_curl_version` + `driver.github_repo` release tag, version-compare via `_pkg_version_lt` | Reads `driver.curl_args` |

All outdated probes are gated behind a single env opt-in:
`UIC_PREF_BREW_LIVECHECK=1`. Off → behavior is presence-only / brew-`outdated`-only,
fast path.

## `pip` driver in detail

| Field | Mechanism |
|---|---|
| Activation | `_pip_ensure_path` → falls back to `_pyenv_ensure_path` → falls back to `python3 -m pip` |
| Outdated | `pip list --outdated --format=json` cached per process; opt-in via `UIC_PREF_BREW_LIVECHECK=1` |
| Update safety | `_pip_update_would_conflict`: dry-run with `--upgrade-strategy only-if-needed`; if pip's resolver would leave another installed package incompatible, the upgrade is **skipped** with a warning. Non-destructive default. |
| Isolation | Optional `driver.isolation: pipx` per target. Each package goes into its own pipx venv. Use only for self-contained CLI tools — pipx targets cannot share imports with other groups. |
| Audit tool | `tools/audit-pip-isolation.py` reports console_scripts vs library shape per package; never writes |

## Outdated detection summary

Behind `UIC_PREF_BREW_LIVECHECK=1`:

- `pkg`: brew, brew-cask, npm, vscode (marketplace API), curl (via `github_repo`), native-pm (per-PM list-upgradable).
- `pip`: `pip list --outdated`.
- `app-bundle`: upstream API per target.
- `git-repo`: local vs remote ref.

No outdated detection (by design):
- `pkg` pyenv (no upstream notion at the version-set level).
- `pkg` ollama (tags are immutable refs).
- `custom-daemon` (observe-only).
- `script-installer`, `build-deps`, `pyenv-brew`, `pip-bootstrap`.
- All config-writer kinds (`setting`, `json-merge`, `brew-analytics`, `git-global`, `zsh-config`, `path-export`, `home-artifact`, `compose-file`, `softwareupdate-schedule`, `brew-unlink`).

## Migration / foreign-install handling

`lib/utils.sh:handle_foreign_install` + per-driver-pair safety probe
(`_assess_migration_safety`). Active in:

- `pkg` install action — any backend, before reinstalling a binary that's
  owned by another PM.
- `desktop_app_handle_unmanaged_cask` — legacy cask handler, refactored to
  call the same helper.
